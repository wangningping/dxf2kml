# DWG2KML 转换器 - 项目总结

## 项目概述

已成功创建 DWG/DXF 到 KML 的转换工具，可以将 AutoCAD 图纸转换为 Google Earth 可用的 KML 格式。

## 项目结构

```
dwg2kml/
├── dwg2kml.py          # 命令行转换工具（主程序）
├── dwg2kml_gui.py      # 图形界面版本
├── install.bat         # Windows 安装脚本
├── install.sh          # Linux/macOS 安装脚本
├── 启动转换器.bat       # Windows 快速启动脚本
├── create_test_dxf.py  # 测试 DXF 文件生成器
├── test.dxf            # 测试 DXF 文件
├── test.kml            # 测试输出 KML 文件
└── README.md           # 使用文档
```

## 功能特性

### 支持的 CAD 实体类型
- LINE（直线）→ LineString
- POLYLINE / LWPOLYLINE（多段线）→ LineString
- CIRCLE（圆）→ Polygon（多边形近似）
- ARC（圆弧）→ LineString
- POINT（点）→ Point
- TEXT / MTEXT（文字）→ Point（带描述）
- INSERT（块引用）→ Point（带描述）

### 坐标转换支持
1. **简单转换**：基于基准经纬度，假设 CAD 单位为米
2. **UTM 转换**：支持 UTM 投影坐标转换（需安装 pyproj）

### 使用方式

#### 命令行模式
```bash
# 基本转换
python dwg2kml.py input.dxf -o output.kml

# 指定基准点
python dwg2kml.py input.dxf -o output.kml --base-lat 39.9 --base-lon 116.4

# 使用 UTM 转换
python dwg2kml.py input.dxf -o output.kml --utm-zone 50

# 批量转换
python dwg2kml.py --batch ./dxf_files -o ./kml_output
```

#### 图形界面模式
```bash
# Windows
启动转换器.bat

# 或直接运行
python dwg2kml_gui.py
```

## 依赖库

- **ezdxf**：DXF 文件解析
- **simplekml**：KML 文件生成
- **pyproj**（可选）：UTM 坐标转换


## 关于 DWG 格式

DWG 是 AutoCAD 的专有二进制格式，无法直接读取。处理 DWG 文件的方法：

1. **在 AutoCAD 中导出**：
   - 打开 DWG 文件
   - 使用 `DXFOUT` 命令导出为 DXF
   - 使用本工具转换

2. **使用 ODA File Converter**：
   - 下载：https://www.opendesign.com/guestfiles/oda_file_converter
   - 批量将 DWG 转换为 DXF
   - 使用本工具转换

## 注意事项

1. **坐标系统**：确保了解 CAD 文件使用的坐标系，选择正确的转换方式
2. **单位**：默认假设 CAD 坐标单位为米
3. **大范围数据**：对于大范围数据，建议使用 UTM 转换以获得更高精度
4. **复杂实体**：3D 实体、曲面等复杂对象暂不支持

## 技术说明

### simplekml 坐标格式
simplekml 的 Point 对象期望 coords 参数为坐标列表：`[(lon, lat)]`
LineString 和 Polygon 期望坐标列表：`[(lon1, lat1), (lon2, lat2), ...]`

### DXF 坐标类型
ezdxf 返回的坐标是 Vec3 类型，有 x, y, z 属性，也可以通过索引访问
