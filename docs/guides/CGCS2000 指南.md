# CGCS2000 坐标系转换指南

## 什么是 CGCS2000？

CGCS2000（China Geodetic Coordinate System 2000，中国大地坐标系 2000）是中国法定的国家大地坐标系，于 2008 年 7 月 1 日正式启用。

## CGCS2000 投影方式

### 1. 高斯 - 克吕格投影（Gauss-Krüger）

中国地形图主要采用高斯 - 克吕格投影，分为 3 度带和 6 度带：

#### 3 度带（推荐，精度更高）
- 带号范围：25-45
- 中央经线计算：`中央经线 = 带号 × 3`
- 适用于大比例尺测图（1:10000 及更大）

#### 6 度带
- 带号范围：13-45
- 中央经线计算：`中央经线 = 带号 × 6 - 3`
- 适用于中小比例尺测图

### 2. 地理坐标系（经纬度）
- 直接使用经纬度表示位置
- EPSG:4490

## 如何确定你的 DWG 文件使用的带号？

### 方法 1：查看 CAD 图纸信息
- 查看图纸标题栏或说明
- 通常标注为"CGCS2000 3 度带 XX 带"或"CGCS2000 6 度带 XX 带"

### 方法 2：根据坐标值估算
- **3 度带**：Y 坐标（东坐标）通常为 6-7 位数，前两位或三位是带号
  - 例如：`39500000` 表示 39 带，实际东坐标为 `500000`
- **6 度带**：Y 坐标前两位是带号
  - 例如：`20500000` 表示 20 带

### 方法 3：根据项目位置
- 使用上表中的城市带号参考
- 或根据经度计算：
  - 3 度带带号 = round(经度 / 3)
  - 6 度带带号 = floor((经度 + 3) / 6)

## 使用 dwg2kml 转换 CGCS2000 文件

### 安装依赖
```bash
pip install ezdxf simplekml pyproj
```

### 转换命令

#### 使用 3 度带（推荐）
```bash
# 北京地区（39 带）
python dwg2kml.py input.dxf -o output.kml --cgcs2000-3deg 39

# 上海地区（41 带）
python dwg2kml.py input.dxf -o output.kml --cgcs2000-3deg 41

# 广州/深圳地区（38 带）
python dwg2kml.py input.dxf -o output.kml --cgcs2000-3deg 38
```

#### 使用 6 度带
```bash
# 北京地区（20 带）
python dwg2kml.py input.dxf -o output.kml --cgcs2000-6deg 20
```

#### 使用地理坐标系（如果 DXF 已经是经纬度）
```bash
python dwg2kml.py input.dxf -o output.kml --cgcs2000-geog
```

#### 批量转换
```bash
# 批量转换整个文件夹
python dwg2kml.py --batch ./dxf_files --cgcs2000-3deg 39
```

## CGCS2000 与 WGS84 的区别

CGCS2000 与 WGS84（GPS 使用的坐标系）非常接近：
- 两者都是地心坐标系
- 椭球参数略有差异（扁率差异约 1/3000000）
- 在同一位置，两者坐标差异通常在厘米级
- 对于一般应用，可以认为两者等价

## 常见问题

### Q: 转换后位置偏移很大怎么办？
A: 检查是否选对了带号。错误的带号会导致几百公里的偏移。

### Q: 如何验证转换是否正确？
A: 在 Google Earth 中打开 KML 文件，查看已知位置（如建筑物、道路交叉口）是否与实际位置吻合。

### Q: DWG 文件可以直接转换吗？
A: DWG 是专有格式，需要先用 AutoCAD 或 ODA File Converter 转换为 DXF。

### Q: 坐标有带号前缀（如 39500000）怎么办？
A: 这种情况需要在 CAD 中先处理，去掉带号前缀，保留实际坐标值（如 500000）。
